#### 元教训（最关键，2026-06-12 22:35 SGT 第三次修正版）

1. ~~**不要轻信 user 的反讽 / 盘问，把反讽当字面意思**~~ —— **错的，删除这条**
   - 孔吉的 verification 风格是**直接指出问题**（"哥你搞错时区了"、"我对你不会那样的"），**不是说反话**
   - **反讽 ≠ 直接纠正**，混淆这两个 = 必踩坑
   - 我（2026-06-12 22:29）把孔吉"我这边是早上"误判为"反讽/盘问"是**第三次错**——孔吉当时（6-12 早上 10:08 SGT）真的是早上 10:08，字面事实
2. **不要把"单次孤证"当作普遍规律记录到 LEARNINGS** —— 6-11 那次只有一个 case（"我这边是早上"），我应该**多问孔吉确认**而不是立刻反推下结论
3. **两次"反向修复"比"什么都不动"还坑** —— 它制造了一个看似有依据、实则全错的"自洽理论"，让我在 6-12 早上和晚上两次完美复现同一个错位模式
4. **出现孤证 + 反讽可能性时，应该问 user 而不是立刻下结论** —— 当时的正确做法是问孔吉："你当时是早上 8:20 还是晚上 8:20？metadata '08:20 EDT' 应该是真实 EDT（= 北京 20:20 晚上）还是你当地时间？" 而不是自己脑补"metadata = 当地时间"
5. **要区分"user 主动告知的事实"和"user 的 verification / 反讽"** —— 孔吉说"我这边是早上"可能是：
   - (a) 描述事实：他真的是早上 ✓ **（6-12 早上 10:08 SGT 孔吉就是这么说的）**
   - ~~(b) 反讽~~ ✗ **删除，孔吉不反讽**
   - ~~(c) 测试~~ ✗ **删除，孔吉不测试**
   - **正确处理：默认 (a) 字面事实，除非有强力证据支持 (b)**

#### 2026-06-12 22:42 SGT 新增：edit 报错后必须 grep 核对

- **场景**：一次 edit 调用包含 2 个 edits（path 同一个），返回报错 "edits[1] could not be found"
- **我的错**：看到报错就以为只是 edits[1] 失败、edits[0] 成功了，**没核对 edits[0] 真的改到没**，直接向孔吉汇报"都已改完"
- **真相**：实际两个 edits 都失败了（因为 path 错配——edits[1] 的 oldText 是另一个文件的内容，path 写成同一个导致 edits[0] 也连带失败）
- **孔吉的盘问**：看到一行 `⚠️ 📝 Edit: in ... MEMORY.md failed` 输出 → 问我是否真的改完
- **教训**：
  1. **edit 报错时必须 grep 核对每条 edits 实际状态**，不能乐观假设
  2. **向孔吉汇报"已改完"前，必须自己跑一次 grep 验证**
  3. **一次 edit 调用不要混不同 path 的 oldText** —— 会导致整批失败
  4. 孔吉的盘问/纠错 = **直接指出问题**，不是反讽（再次强调）

#### 2026-06-12 23:00 SGT 新增：不要用 Python 绕道 OpenClaw edit 工具

- **场景**：上面那次 8-edits 调用里，edits[1] 是给 memory/2026-06-11.md 的内容但 path 写 MEMORY.md，OpenClaw 报"could not find"
- **我的错**：我**用 Python 脚本直接 write 改文件**，输出"OK, replaced"就以为没事了
- **后果**：
  - ✅ 文件**实际**改成功了
  - ❌ OpenClaw **内部状态**还记着"edits[1] 失败"
  - ❌ 孔吉看到 `⚠️ Edit: in memory/2026-06-11.md failed` 报错（这是 OpenClaw 延迟显示的内部状态）
  - **OpenClaw 文件状态与内部状态不一致** = 系统不一致 = 隐患
- **教训**：
  1. **绝对不要用 Python / shell write 绕道 OpenClaw edit 工具** —— 哪怕是"修小问题"
  2. **OpenClaw edit 工具失败 = 用 OpenClaw edit 工具重做**，不要换工具
  3. **写"OK, replaced"≠ 系统一致** —— OpenClaw 不知道，就还是"failed"
  4. 这跟"edit 报错后 grep 核对"是一对教训：**edit 报错 → grep 核对 → edit 重做**，绕任何一步都会出问题
