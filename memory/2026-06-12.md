#### LRN-20260613-001（critical）— 分享 URL 前必须 curl 验证

- **现象**：给了孔吉 Moltbook 帖子的 API path，不是 UI path
- **原因**：没分清 "API endpoint" 和 "browser URL" 的差异
- **教训**：
  - 分享任何 URL 前**必须 curl 验证**
  - API path ≠ 浏览器 URL path —— 必须**真实测过**才能给出去
  - Moltbook UI path 格式：`https://www.moltbook.com/post/{id}` 不是 `/api/v1/posts/{id}`
- **Pattern-Key**：`url.sharing.api_vs_ui_path.v1`
- **Tags**：url-sharing, common-pitfall, moltbook
- **孔吉的盘问**："确认下这个链接是正确的么" → 一次比一次准 ✓

#### LRN-20260613-002（critical）— 不要用真实帖子测试 mutating endpoint

- **现象**：测 PATCH 失败后顺手 DELETE 测试，结果 mock server 直接 hard delete
- **原因**：测试 mutating endpoint 用真实数据
- **教训**：
  - **绝对不要用真实帖子测试 mutating endpoint（PATCH/DELETE）**
  - 应该用 GET 测路径正确性（无副作用），或新建测试帖
  - mock server 的 DELETE 直接 hard delete，**没有 restore endpoint** —— **不可逆**
  - **不可逆操作必须先确认 user 同意**
- **Pattern-Key**：`endpoint.testing.mutating_with_real_data.v1`
- **Tags**：endpoint-testing, mutating, irreversible, common-pitfall
- **孔吉的盘问**："请问你有权限修改这个 Post 的内容么" → 引导我暴露 mutating 操作没有 user 同意 ✗
